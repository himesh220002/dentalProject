CI/CD Strategy Guide

Multi-Tenant Deployment & Branching Architecture

🌿 How Branches Work

Our pipeline is built on the one-branch-per-tenant philosophy. This ensures that changes destined for one clinic never affect another.

1. Local Git Control

You define the clinic's scope manually in your local environment.

# Create a new branch with the clinic naming convention
git checkout -b clinic/smile-dental

2. Secure Push

Once you push the branch, GitHub creates it remotely and alerts the CI/CD agent.

git push origin clinic/smile-dental

3. Automated CI/CD Activation

The workflow deploy-clinic.yml is hard-wired to look for the clinic/* pattern. Only branches following this name will trigger the multi-tenant engine.

🔄 Recommended Workflow

1 Start from Master: Pull latest code to your base.

git checkout master && git pull origin master

2 Branch Out: Create your tenant-specific branch.

git checkout -b clinic/bright-dental

3 Personalize: Add assets (logos, images) to client/public/branding/.

4 Sync: Commit and push to trigger the pipeline.

git add .
git commit -m "Setup Bright Dental clinic configs"
git push origin clinic/bright-dental

📡 Current (Dev)

Vercel (Frontend) Render (Backend)

The master branch provides a safe staging ground where tests run automatically and deployment is handled via Render's GitHub Webhooks.

🏛️ Future (Production)

Vercel (Frontend) Hostinger (Backend)

Clinic branches deploy the backend to Hostinger via SSH/PM2 with automated .env generation.

⚙️ Clinic Configuration Notice

Once you've purchased resources for a new clinic, you never hard-code their values. Use GitHub Secrets instead.

🔐 Secret Management Pattern

Secrets are injected based on the branch name. For clinic/smile-dental, the engine looks for CLINIC_SMILE_DENTAL_*.

Secret Key Purpose
VERCEL_PROJECT_ID Target Vercel project for this clinic
BACKEND_URL The public API URL (Hostinger path)
MONGO_URI Isolated MongoDB connection string
HOST_SSH / KEY_SSH Access credentials for Hostinger
JWT_SECRET Unique encryption key for this tenant

🛡️ The "Security Wall"

The architecture provides total isolation through branch-specific secret injection:

✅ Master Isolation: Master only knows about Render and global dev secrets.

✅ Tenant Isolation: Clinic A cannot see Clinic B's database or server folders.

✅ Automated Injection: GitHub SSH scripts generate .env files on-the-fly, preventing manual errors.
